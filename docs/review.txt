以下是按阻断程度排序的评审意见。结论先说狠一点：这次实现有后端核心雏形，但还没有达到 [message-queue-admin-design.zh-CN.md](D:/code/dbx/docs/message-queue-admin-design.zh-CN.md) 描述的“已接入、可用、可发布”状态。

1. **P0 桌面端和 Web 端的 `mq-admin` 开关没有闭合，MQ 命令/路由默认不会进构建。**  
   `src-tauri` 的默认 feature 只有 `duckdb-bundled`，且依赖 `dbx-core` 时关闭了默认 feature：[src-tauri/Cargo.toml](D:/code/dbx/src-tauri/Cargo.toml:16)、[src-tauri/Cargo.toml](D:/code/dbx/src-tauri/Cargo.toml:58)。但 MQ 命令注册又全部挂在 `#[cfg(feature = "mq-admin")]` 下：[src-tauri/src/lib.rs](D:/code/dbx/src-tauri/src/lib.rs:496)。Web 侧也一样，路由模块和 `/mq/...` 路由都被本 crate 的 `mq-admin` feature gate 包住：[crates/dbx-web/src/routes/mod.rs](D:/code/dbx/crates/dbx-web/src/routes/mod.rs:12)、[crates/dbx-web/src/main.rs](D:/code/dbx/crates/dbx-web/src/main.rs:279)，但 [crates/dbx-web/Cargo.toml](D:/code/dbx/crates/dbx-web/Cargo.toml:1) 没有声明这个 feature。建议先补齐 `mq-admin = ["dbx-core/mq-admin"]`，并明确是否进 `default`。

2. **P0 前端没有把 MQ 控制台挂到主内容区，`mode="mq"` 会没有渲染分支。**  
   类型里已经加了 `"mq"`：[apps/desktop/src/types/database.ts](D:/code/dbx/apps/desktop/src/types/database.ts:54)、[apps/desktop/src/types/database.ts](D:/code/dbx/apps/desktop/src/types/database.ts:474)，但 [ContentArea.vue](D:/code/dbx/apps/desktop/src/components/layout/ContentArea.vue:427) 只有 query/data/redis/etcd/mongo/objects/structure/users 等分支，直到 [ContentArea.vue](D:/code/dbx/apps/desktop/src/components/layout/ContentArea.vue:857) 都没有 `mq`。新组件自己也写着还需要添加 `mode === 'mq'` 分支：[MqAdminConsole.vue](D:/code/dbx/apps/desktop/src/components/mq/MqAdminConsole.vue:14)。所以即使创建出 MQ tab，用户也看不到控制台。

3. **P1 用户没有正常路径创建、发现或打开 MQ 连接。**  
   连接弹窗的 `driverProfiles` 从 [ConnectionDialog.vue](D:/code/dbx/apps/desktop/src/components/connection/ConnectionDialog.vue:325) 开始，没有 Pulsar/MQ profile，也没有 `adminUrl`、`systemKind`、auth 等 `external_config` 表单；提交函数只是规范化通用数据库字段：[ConnectionDialog.vue](D:/code/dbx/apps/desktop/src/components/connection/ConnectionDialog.vue:1054)。侧边栏连接打开逻辑只处理 redis/etcd/mongodb/elasticsearch 等：[TreeItem.vue](D:/code/dbx/apps/desktop/src/components/sidebar/TreeItem.vue:328)。驱动 manifest 也没有 `"dbType": "mq"`，目前能看到 `etcd` 等条目但无 MQ：[database-drivers.manifest.json](D:/code/dbx/crates/dbx-core/assets/database-drivers.manifest.json:1373)。这会让能力发现、连接创建和树入口整条链都断掉。

4. **P1 MQ 认证信息有明文落盘风险。**  
   MQ 配置把 `MqAuth` 放在 `ConnectionConfig.external_config`：[config.rs](D:/code/dbx/crates/dbx-core/src/mq/config.rs:14)，而 `MqAuth` 包含 token、basic password、OAuth2 client_secret：[auth.rs](D:/code/dbx/crates/dbx-core/src/mq/auth.rs:20)、[auth.rs](D:/code/dbx/crates/dbx-core/src/mq/auth.rs:22)、[auth.rs](D:/code/dbx/crates/dbx-core/src/mq/auth.rs:29)。现有 secret 保存只处理固定字段，如 `password`、`redis_sentinel_password`、`connection_string`：[connection_secrets.rs](D:/code/dbx/crates/dbx-core/src/connection_secrets.rs:75)、[connection_secrets.rs](D:/code/dbx/crates/dbx-core/src/connection_secrets.rs:80)、[connection_secrets.rs](D:/code/dbx/crates/dbx-core/src/connection_secrets.rs:81)，清洗函数也没有处理 `external_config`：[connection_secrets.rs](D:/code/dbx/crates/dbx-core/src/connection_secrets.rs:382)。建议把 MQ auth 拆成专用 secret key，或给 `external_config` 引入结构化敏感字段迁移/清洗。

5. **P1 Web HTTP API 路径少了 `/api` 前缀。**  
   Web 服务把所有 API 挂在 `/api` 下：[crates/dbx-web/src/main.rs](D:/code/dbx/crates/dbx-web/src/main.rs:415)，实际 MQ 路径应是 `/api/mq/...`。但前端 HTTP wrapper 调的是 `/mq/...`：[mq-http.ts](D:/code/dbx/apps/desktop/src/lib/mq-http.ts:44)、[mq-http.ts](D:/code/dbx/apps/desktop/src/lib/mq-http.ts:188)。Web 模式下这些请求会 404。

6. **P2 UI 仍是半成品，不满足文档里的“完整可用 MQ Admin UI”。**  
   当前主控台只接了基础 tab，且组件注释还把集成工作列为待办：[MqAdminConsole.vue](D:/code/dbx/apps/desktop/src/components/mq/MqAdminConsole.vue:14)。从子组件看，主要覆盖 tenants/namespaces/topics/subscriptions/monitoring，缺少文档要求的 raw API 面板、策略/权限的完整编辑面板、只读模式下按钮禁用/提示等。后端边界有只读检查是好事：[mq_cmd.rs](D:/code/dbx/src-tauri/src/commands/mq_cmd.rs:48)、[mq_cmd.rs](D:/code/dbx/src-tauri/src/commands/mq_cmd.rs:412)、[mq.rs](D:/code/dbx/crates/dbx-web/src/routes/mq.rs:660)，但前端 destructive 操作仍应禁用并解释原因。

7. **P2 Pulsar Admin URL path segment 没做编码，名称里有特殊字符会打错 endpoint。**  
   现在大量路径用裸 `format!("/admin/v2/.../{tenant}/{namespace}/...")` 拼接：[pulsar_version.rs](D:/code/dbx/crates/dbx-core/src/mq/adapters/pulsar_version.rs:160)，topic path 也是直接拼字符串：[types.rs](D:/code/dbx/crates/dbx-core/src/mq/types.rs:188)。tenant、namespace、topic、subscription、role 如果包含空格、`/`、`?`、`#` 等保留字符，会导致请求失败，甚至改变目标路径。建议对每个 path segment 单独 percent-encode，保留有意的 Pulsar domain/path 分隔。

8. **P2 只读保护没有下沉到共享 service 层，和设计有偏差。**  
   service 层的 mutating 方法直接调用 adapter，例如 create/delete/set/grant/revoke：[service.rs](D:/code/dbx/crates/dbx-core/src/mq/service.rs:31)、[service.rs](D:/code/dbx/crates/dbx-core/src/mq/service.rs:317)、[service.rs](D:/code/dbx/crates/dbx-core/src/mq/service.rs:361)。当前 Tauri/Web 边界做了拦截，但设计明确强调共享 core service 作为两端复用点；后续如果新增调用方直接走 service，会绕过只读保护。建议把 writable guard 放进 service，边界层可以保留作为双保险。

9. **P3 前端 MQ API 绕开了既有 `forward()` 机制，架构风格不一致。**  
   文档要求沿用 `apps/desktop/src/lib/api.ts` 的 `forward()`，现有 API 层也以 [api.ts](D:/code/dbx/apps/desktop/src/lib/api.ts:23) 为统一入口。但新文件手动判断 Tauri/HTTP：[mq-api.ts](D:/code/dbx/apps/desktop/src/lib/mq-api.ts:2)、[mq-api.ts](D:/code/dbx/apps/desktop/src/lib/mq-api.ts:9)。这会让运行时选择、命名映射、错误处理和未来维护出现第二套规则。建议把 MQ 方法并入 `api.ts`/Backend contract，或至少复用同一 forward 机制。
